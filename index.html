CSS Fundamentals

CSS Syntax Structure

Understanding how CSS rules are constructed is essential for writing effective stylesheets.

/* Selector targets the element */
        selector {
            property: value; /* Declaration */
            another-property: value;
        }

Gotchas & Best Practices:

  • Always end declarations with semicolons, even the last one in a rule.
  • Use consistent indentation (2 or 4 spaces).
  • Group related properties together for better organization.
  • Use meaningful class names that describe purpose, not appearance.

CSS Specificity

Specificity determines which styles are applied when multiple rules target the same element.

Specificity Weight (from highest to lowest):

  • Inline styles: style="color: red;" (1000 points)
  • IDs: #header (100 points)
  • Classes, attributes, pseudo-classes: .nav, [type="text"], :hover (10 points)
  • Elements and pseudo-elements: div, ::before (1 point)

<!-- end list -->

/* Specificity: 001 (1 element) */
        p { color: blue; }

        /* Specificity: 011 (1 class + 1 element) */
        p.highlight { color: red; }

        /* Specificity: 101 (1 ID + 1 element) */
        p#intro { color: green; }

Gotchas & Best Practices:

  • Avoid using !important unless absolutely necessary.
  • Keep specificity as low as possible while achieving desired results.
  • Use classes more than IDs for styling (IDs should be for JavaScript hooks).
  • Understanding specificity prevents many styling conflicts.

Selectors

Basic Selectors

Universal Selector

Targets every element on the page.

* {
            box-sizing: border-box; /* Apply to all elements */
            margin: 0;
            padding: 0;
        }

Element Selectors

Target elements by their HTML tag name.

h1 { font-size: 2rem; }
        p { line-height: 1.6; }
        img { max-width: 100%; height: auto; }

Class Selectors

Target elements with specific class attributes.

.container { max-width: 1200px; margin: 0 auto; }
        .btn { padding: 10px 20px; border: none; cursor: pointer; }
        .btn-primary { background-color: #007bff; color: white; }

ID Selectors

Target a unique element with a specific ID.

#header { background-color: #f8f9fa; }
        #sidebar { width: 300px; float: left; }

Attribute Selectors

Target elements based on their attributes.

/* Has attribute */
        [disabled] { opacity: 0.5; }

        /* Exact match */
        [type="email"] { border-color: #28a745; }

        /* Contains word */
        [class~="active"] { font-weight: bold; }

        /* Starts with */
        [href^="https://"] { color: green; }

        /* Ends with */
        [src$=".jpg"] { border: 2px solid #ccc; }

        /* Contains substring */
        [alt*="logo"] { width: 200px; }

Pseudo-Classes

Target elements in specific states.

/* Link states */
        a:link { color: blue; }
        a:visited { color: purple; }
        a:hover { color: red; text-decoration: underline; }
        a:active { color: darkred; }

        /* Form states */
        input:focus { outline: 2px solid #007bff; }
        input:valid { border-color: green; }
        input:invalid { border-color: red; }
        input:disabled { background-color: #f5f5f5; }

        /* Structural pseudo-classes */
        li:first-child { margin-top: 0; }
        li:last-child { margin-bottom: 0; }
        tr:nth-child(even) { background-color: #f8f9fa; }
        tr:nth-child(odd) { background-color: white; }
        p:nth-of-type(3) { font-weight: bold; }

Pseudo-Elements

Create and style virtual elements.

/* Add content before/after elements */
        .quote::before { content: '"'; font-size: 1.2em; }
        .quote::after { content: '"'; font-size: 1.2em; }

        /* Style first letter/line */
        p::first-letter { font-size: 2em; float: left; }
        p::first-line { font-weight: bold; }

        /* Style selected text */
        ::selection { background-color: #007bff; color: white; }

Combinators

Define relationships between elements.

/* Descendant combinator (space) */
        .nav a { color: #333; } /* All anchors inside .nav */

        /* Child combinator (>) */
        .menu > li { display: inline-block; } /* Direct children only */

        /* Adjacent sibling combinator (+) */
        h1 + p { margin-top: 0; } /* First paragraph after h1 */

        /* General sibling combinator (~) */
        h1 ~ p { color: #666; } /* All paragraphs after h1 */

Gotchas & Best Practices:

  • Use descendant selectors carefully to avoid overly specific rules.
  • Pseudo-classes like :hover work better on interactive elements.
  • Remember that :visited has security limitations in modern browsers.
  • ::before and ::after pseudo-elements require the content property to display.

Box Model & Spacing

Box Model Basics

Every element is a rectangular box with content, padding, border, and margin.

.box {
            width: 300px;           /* Content area width */
            height: 200px;          /* Content area height */
            padding: 20px;          /* Space between content and border */
            border: 2px solid #333; /* Border around padding */
            margin: 10px;           /* Space outside border */
        }

Box-Sizing Property

Controls how width and height are calculated.

/* Default behavior - padding and border add to width */
        .content-box { box-sizing: content-box; }

        /* Modern approach - padding and border included in width */
        .border-box { box-sizing: border-box; }

        /* Apply to all elements (recommended) */
        * { box-sizing: border-box; }

Margin Properties

Control spacing outside the element’s border.

/* Individual sides */
        .element {
            margin-top: 10px;
            margin-right: 15px;
            margin-bottom: 10px;
            margin-left: 15px;
        }

        /* Shorthand values */
        .shorthand-four { margin: 10px 15px 10px 15px; } /* top right bottom left */
        .shorthand-three { margin: 10px 15px 20px; }     /* top horizontal bottom */
        .shorthand-two { margin: 10px 15px; }            /* vertical horizontal */
        .shorthand-one { margin: 10px; }                 /* all sides */

        /* Auto margins for centering */
        .center { margin: 0 auto; width: 300px; }

        /* Negative margins */
        .overlap { margin-top: -10px; } /* Pull element up */

Padding Properties

Control spacing inside the element’s border.

/* Individual sides */
        .padded {
            padding-top: 20px;
            padding-right: 30px;
            padding-bottom: 20px;
            padding-left: 30px;
        }

        /* Shorthand (same syntax as margin) */
        .padding-shorthand { padding: 20px 30px; } /* vertical horizontal */

        /* Percentage padding (relative to parent width) */
        .responsive-padding { padding: 5% 10%; }

Border Properties

Define the border around an element.

/* Basic border */
        .basic-border { border: 2px solid #333; }

        /* Individual properties */
        .detailed-border {
            border-width: 2px;      /* or thin, medium, thick */
            border-style: solid;    /* or dotted, dashed, double, groove, ridge, inset, outset */
            border-color: #333;
        }

        /* Individual sides */
        .custom-sides {
            border-top: 1px solid #ccc;
            border-right: none;
            border-bottom: 2px dashed #999;
            border-left: 3px solid #007bff;
        }

        /* Border radius for rounded corners */
        .rounded {
            border-radius: 5px;              /* All corners */
            border-radius: 10px 5px;         /* horizontal vertical */
            border-radius: 10px 5px 15px 20px; /* top-left top-right bottom-right bottom-left */
        }

        /* Individual corner radius */
        .custom-corners {
            border-top-left-radius: 10px;
            border-top-right-radius: 5px;
            border-bottom-right-radius: 15px;
            border-bottom-left-radius: 20px;
        }

Outline Properties

Similar to border but doesn’t affect layout.

.outlined {
            outline: 2px solid red; /* Basic outline */
            outline-offset: 5px;    /* Distance from border */
        }

        /* Useful for focus states */
        button:focus {
            outline: 2px solid #007bff;
            outline-offset: 2px;
        }

Gotchas & Best Practices:

  • Margin collapse occurs when vertical margins of adjacent elements combine.
  • Use box-sizing: border-box as a global reset for more predictable sizing.
  • Percentage padding/margin is calculated from parent’s width, not height.
  • Outline doesn’t take up space in the layout, making it perfect for focus indicators.
  • Auto margins only work for horizontal centering on block elements with defined width.

Layout Systems

Display Property

Controls how an element generates its box model.

/* Block elements take full width */
        .block { display: block; }
        /* Inline elements flow with text */
        .inline { display: inline; }
        /* Inline-block combines both behaviors */
        .inline-block { display: inline-block; width: 100px; height: 50px; }
        /* Hide elements */
        .hidden { display: none; }
        /* Removes from layout flow */
        .invisible { visibility: hidden; }
        /* Keeps space, hides content */
        /* Table display values */
        .table { display: table; }
        .table-row { display: table-row; }
        .table-cell { display: table-cell; vertical-align: middle; }

Float and Clear

Traditional layout method, now mostly replaced by Flexbox and Grid.

/* Float elements */
        .float-left { float: left; width: 300px; }
        .float-right { float: right; width: 200px; }
        /* Clear floats */
        .clear-both { clear: both; }
        .clear-left { clear: left; }
        .clear-right { clear: right; }
        /* Clearfix hack for containing floats */
        .clearfix::after {
            content: "";
            display: table;
            clear: both;
        }

Flexbox Layout

Modern one-dimensional layout system for arranging elements in rows or columns.

Flex Container Properties

.flex-container {
            display: flex; /* or inline-flex */
            /* Direction of main axis */
            flex-direction: row; /* default - left to right */
            flex-direction: row-reverse; /* right to left */
            flex-direction: column; /* top to bottom */
            flex-direction: column-reverse; /* bottom to top */
            /* Wrapping behavior */
            flex-wrap: nowrap; /* default - single line */
            flex-wrap: wrap; /* wrap to new lines */
            flex-wrap: wrap-reverse; /* wrap in reverse order */
            /* Shorthand for direction and wrap */
            flex-flow: row wrap;
            /* Alignment along main axis */
            justify-content: flex-start; /* default - start of container */
            justify-content: flex-end; /* end of container */
            justify-content: center; /* center of container */
            justify-content: space-between; /* even distribution, no edge spacing */
            justify-content: space-around; /* even distribution with edge spacing */
            justify-content: space-evenly; /* equal space everywhere */
            /* Alignment along cross axis */
            align-items: stretch; /* default - fill container height */
            align-items: flex-start; /* align to start */
            align-items: flex-end; /* align to end */
            align-items: center; /* center alignment */
            align-items: baseline; /* align baselines */
            /* Multi-line cross-axis alignment */
            align-content: stretch; /* default */
            align-content: flex-start;
            align-content: flex-end;
            align-content: center;
            align-content: space-between;
            align-content: space-around;
            /* Gap between items */
            gap: 20px; /* both row and column gap */
            row-gap: 10px; /* gap between rows */
            column-gap: 15px; /* gap between columns */
        }

Flex Item Properties

.flex-item {
            /* Growth factor */
            flex-grow: 0; /* default - don't grow */
            flex-grow: 1; /* grow to fill available space */
            flex-grow: 2; /* grow twice as much as items with flex-grow: 1 */
            /* Shrink factor */
            flex-shrink: 1; /* default - can shrink */
            flex-shrink: 0; /* don't shrink */
            /* Base size before free space distribution */
            flex-basis: auto; /* default - use content size */
            flex-basis: 200px; /* specific size */
            flex-basis: 0; /* ignore content size */
            /* Shorthand for grow, shrink, basis */
            flex: 0 1 auto; /* default */
            flex: 1; /* shorthand for 1 1 0 */
            flex: none; /* shorthand for 0 0 auto */
            flex: auto; /* shorthand for 1 1 auto */
            /* Individual cross-axis alignment */
            align-self: auto; /* default - inherit from container */
            align-self: flex-start;
            align-self: flex-end;
            align-self: center;
            align-self: baseline;
            align-self: stretch;
            /* Change visual order without affecting HTML */
            order: 0; /* default - original order */
            order: 1; /* move to end */
            order: -1; /* move to beginning */
        }

Common Flexbox Patterns

/* Center content both ways */
        .flex-center {
            display: flex;
            justify-content: center;
            align-items: center;
            height: 100vh; /* full viewport height */
        }

        /* Navigation bar */
        .navbar {
            display: flex;
            justify-content: space-between;
            align-items: center;
            padding: 1rem;
        }

        /* Card layout with equal height */
        .card-container {
            display: flex;
            gap: 1rem;
        }

        .card {
            flex: 1;
        }

        /* Equal width cards */
        /* Sticky footer */
        .page-layout {
            display: flex;
            flex-direction: column;
            min-height: 100vh;
        }

        .main-content {
            flex: 1;
        }

CSS Grid Layout

Modern two-dimensional layout system for complex layouts.

Grid Container Properties

.grid-container {
            display: grid; /* or inline-grid */
            /* Define columns */
            grid-template-columns: 200px 1fr 100px; /* fixed, flexible, fixed */
            grid-template-columns: repeat(3, 1fr); /* three equal columns */
            grid-template-columns: repeat(auto-fit, 250px); /* responsive columns */
            grid-template-columns: minmax(200px, 1fr) 2fr; /* minimum and maximum */
            /* Define rows */
            grid-template-rows: 100px auto 50px; /* fixed, content-based, fixed */
            grid-template-rows: repeat(3, 150px); /* three equal rows */
            /* Define named grid areas */
            grid-template-areas:
                "header header header"
                "sidebar main main"
                "footer footer footer";
            /* Shorthand for all template properties */
            grid-template:
                "header header header" 100px
                "sidebar main main" 1fr
                "footer footer footer" 50px
                / 200px 1fr 100px;
            /* Gap between grid items */
            gap: 20px; /* both row and column gap */
            row-gap: 10px; /* gap between rows */
            column-gap: 15px; /* gap between columns */
            /* Alignment of grid items */
            justify-items: stretch; /* default - fill cell width */
            justify-items: start; /* align to cell start */
            justify-items: end; /* align to cell end */
            justify-items: center; /* center in cell */
            align-items: stretch; /* default - fill cell height */
            align-items: start; /* align to cell top */
            align-items: end; /* align to cell bottom */
            align-items: center; /* center in cell */
            /* Shorthand for item alignment */
            place-items: center; /* center both ways */
            place-items: start end; /* align-items start, justify-items end */
            /* Alignment of entire grid within container */
            justify-content: start; /* align grid to container start */
            justify-content: end; /* align grid to container end */
            justify-content: center; /* center grid in container */
            justify-content: stretch; /* stretch to fill container */
        }

Grid Item Properties

.grid-item {
            /* Position item in grid */
            grid-column-start: 2; /* start at line 2 */
            grid-column-end: 4; /* end at line 4 */
            grid-row-start: 1;
            grid-row-end: 2;
            /* Shorthand for position */
            grid-column: 2 / 4;
            grid-row: 1 / 2;
            /* Shorthand for position and span */
            grid-column: 2 / span 2; /* span 2 columns */
            /* Position with named areas */
            grid-area: header; /* place item in 'header' area */
            /* Individual alignment */
            justify-self: stretch; /* fill cell width */
            justify-self: start;
            justify-self: end;
            justify-self: center;
            align-self: stretch; /* fill cell height */
            align-self: start;
            align-self: end;
            align-self: center;
            /* Shorthand for self-alignment */
            place-self: center; /* center both ways */
        }

Common Grid Patterns

/* Responsive 12-column grid */
        .grid-12 {
            display: grid;
            grid-template-columns: repeat(12, 1fr);
            gap: 1rem;
        }

        /* Holy Grail Layout */
        .holy-grail {
            display: grid;
            grid-template-columns: 200px 1fr 200px;
            grid-template-rows: auto 1fr auto;
            grid-template-areas:
                "header header header"
                "left main right"
                "footer footer footer";
            min-height: 100vh;
        }

        .holy-grail-header { grid-area: header; }
        .holy-grail-left { grid-area: left; }
        .holy-grail-main { grid-area: main; }
        .holy-grail-right { grid-area: right; }
        .holy-grail-footer { grid-area: footer; }

Gotchas & Best Practices:

  • Flexbox is for one-dimensional layouts (rows or columns); Grid is for two-dimensional layouts.
  • Don’t use floats for main page layout in new projects.
  • Use display: inline-block when you need block-level properties (width, height) on inline elements.
  • display: none removes the element completely from the render tree; visibility: hidden hides it but keeps its space.
  • Grid auto-fit and auto-fill are powerful for responsive design.

Typography

Font Properties

Control the font family, size, weight, and style.

/* Font family */
        .sans-serif { font-family: "Helvetica Neue", Arial, sans-serif; }
        .serif { font-family: "Georgia", serif; }
        .monospace { font-family: "Courier New", monospace; }

        /* Font size */
        .text-lg { font-size: 1.5rem; }
        .text-sm { font-size: 0.875rem; }
        /* Use `rem` for accessibility (scales with browser settings) */

        /* Font weight */
        .font-bold { font-weight: bold; } /* or 700 */
        .font-light { font-weight: 300; }

        /* Font style */
        .italic { font-style: italic; }

        /* Shorthand property */
        .font-shorthand { font: italic bold 1rem/1.5 "Helvetica Neue", sans-serif; }

Text Properties

Control text alignment, decoration, spacing, and transformation.

/* Text alignment */
        .text-center { text-align: center; }
        .text-left { text-align: left; }
        .text-right { text-align: right; }
        .text-justify { text-align: justify; }

        /* Text decoration */
        .underline { text-decoration: underline; }
        .no-underline { text-decoration: none; }
        .line-through { text-decoration: line-through; }
        /* Modern syntax with properties */
        .underline-red {
            text-decoration-line: underline;
            text-decoration-color: red;
            text-decoration-style: wavy;
            text-decoration-thickness: 2px;
        }

        /* Text transform */
        .uppercase { text-transform: uppercase; }
        .lowercase { text-transform: lowercase; }
        .capitalize { text-transform: capitalize; }

        /* Letter and word spacing */
        .letter-spacing { letter-spacing: 2px; }
        .word-spacing { word-spacing: 5px; }

        /* Line height */
        .line-height-tight { line-height: 1.2; }
        .line-height-loose { line-height: 1.8; }
        /* Use unitless values for better scaling */

        /* White space control */
        .nowrap { white-space: nowrap; }
        .pre { white-space: pre; } /* preserves whitespace and line breaks */

        /* Text overflow */
        .ellipsis {
            overflow: hidden;
            white-space: nowrap;
            text-overflow: ellipsis;
        }

Typography Best Practices

  • Readability: Choose fonts and line heights that are easy to read.
  • Hierarchy: Use font size, weight, and color to create a clear visual hierarchy.
  • Accessibility: Ensure sufficient contrast and use relative units like rem or em.
  • Web Fonts: Use @font-face or a service like Google Fonts to load custom fonts.

Colors & Backgrounds

Color Properties

Control the color of text, borders, and other elements.

/* Color names */
        .red-text { color: red; }
        /* HEX codes */
        .blue-text { color: #007bff; }
        /* RGB values */
        .green-text { color: rgb(40, 167, 69); }
        /* RGBA values (A for alpha/opacity) */
        .semi-transparent { color: rgba(0, 0, 0, 0.5); }
        /* HSL values (Hue, Saturation, Lightness) */
        .yellow-text { color: hsl(50, 100%, 50%); }
        /* HSLA values (A for alpha/opacity) */
        .semi-transparent-yellow { color: hsla(50, 100%, 50%, 0.8); }

Background Properties

Define the background of an element.

/* Background color */
        .bg-primary { background-color: #007bff; }

        /* Background image */
        .bg-image { background-image: url("image.jpg"); }

        /* Background repeat */
        .no-repeat { background-repeat: no-repeat; }
        .repeat-x { background-repeat: repeat-x; }

        /* Background position */
        .bg-center { background-position: center center; }
        .bg-top-right { background-position: top right; }
        .bg-position-custom { background-position: 50% 100px; }

        /* Background size */
        .bg-cover { background-size: cover; } /* fills the container */
        .bg-contain { background-size: contain; } /* fits inside the container */
        .bg-size-custom { background-size: 200px auto; }

        /* Background attachment */
        .fixed-bg { background-attachment: fixed; } /* scrolls with viewport */
        .scroll-bg { background-attachment: scroll; } /* scrolls with element (default) */

        /* Shorthand property */
        .bg-shorthand {
            background: #f8f9fa url("image.jpg") no-repeat center/cover fixed;
        }

        /* Multiple backgrounds */
        .multiple-bg {
            background:
                url("layer1.png") no-repeat top left,
                url("layer2.png") no-repeat bottom right;
        }

Gradients

Create smooth color transitions as a background.

/* Linear gradient */
        .linear-gradient {
            background-image: linear-gradient(to right, #007bff, #28a745);
            background-image: linear-gradient(45deg, #007bff, #28a745);
        }

        /* Radial gradient */
        .radial-gradient {
            background-image: radial-gradient(circle, #007bff, #28a745);
        }

Gotchas & Best Practices:

  • Use hex codes or HSL for better readability and a smaller file size.
  • Always provide a fallback background-color for background-image.
  • Be mindful of color contrast for accessibility, especially with text over a background image.

Positioning

Position Property

Control the position of an element.

/* Default - positioned according to the normal flow */
        .static { position: static; }

        /* Relative to its normal position */
        .relative { position: relative; top: 10px; left: 20px; }
        /* `relative` elements are still part of the normal document flow */

        /* Relative to the nearest positioned ancestor */
        .absolute { position: absolute; top: 0; right: 0; }
        /* `absolute` elements are removed from the normal document flow */

        /* Relative to the viewport */
        .fixed { position: fixed; top: 0; left: 0; width: 100%; }
        /* Stays in the same position when the page is scrolled */

        /* Hybrid of `relative` and `fixed` */
        .sticky { position: sticky; top: 0; }
        /* Behaves as `relative` until it hits a threshold, then becomes `fixed` */

Z-Index

Control the stacking order of positioned elements.

/* Higher number means it's "on top" */
        .top-layer { position: relative; z-index: 100; }
        .bottom-layer { position: relative; z-index: 1; }

Gotchas & Best Practices:

  • z-index only works on elements with a position value other than static.
  • sticky requires a top, right, bottom, or left property to work.
  • When using absolute positioning, ensure the parent element has a position of relative, absolute, fixed, or sticky.

Transforms & Animations

Transforms

Apply 2D or 3D transformations to an element.

/* Translate (move) */
        .translate { transform: translate(50px, 100px); }
        .translate-x { transform: translateX(50px); }
        .translate-y { transform: translateY(100px); }

        /* Rotate */
        .rotate-2d { transform: rotate(45deg); }
        .rotate-3d { transform: rotateX(45deg); }

        /* Scale (resize) */
        .scale { transform: scale(1.2); }
        .scale-x { transform: scaleX(1.5); }
        .scale-y { transform: scaleY(0.5); }

        /* Skew (distort) */
        .skew-x { transform: skewX(15deg); }
        .skew-y { transform: skewY(10deg); }

        /* Multiple transforms */
        .multiple { transform: rotate(10deg) scale(1.1) translateX(20px); }

        /* Transform origin */
        .origin-bottom-right { transform-origin: bottom right; }

Transitions

Create smooth changes between property values.

.box-transition {
            /* The property to transition */
            transition-property: background-color;
            /* Duration of the transition */
            transition-duration: 0.3s;
            /* Timing function (easing) */
            transition-timing-function: ease-in-out;
            /* Delay before the transition starts */
            transition-delay: 0.1s;
        }

        /* Shorthand property */
        .box-shorthand { transition: background-color 0.3s ease-in-out; }

        /* Multiple properties */
        .multiple-transitions { transition: background-color 0.3s ease, transform 0.2s ease-out; }

        /* Example: Hover effect */
        .button {
            background-color: blue;
            transition: background-color 0.3s ease;
        }

        .button:hover {
            background-color: darkblue;
        }

Animations

Create complex, multi-step animations.

Keyframes

/* Define the animation steps */
        @keyframes fadeIn {
            from { opacity: 0; }
            to { opacity: 1; }
        }

        @keyframes slideIn {
            0%   { transform: translateX(-100%); }
            50%  { transform: translateX(20px); }
            100% { transform: translateX(0); }
        }

Animation Properties

.animated-element {
            /* Name of the animation to use */
            animation-name: fadeIn;
            /* Duration of one animation cycle */
            animation-duration: 1s;
            /* Timing function */
            animation-timing-function: ease-in;
            /* Delay before animation starts */
            animation-delay: 0.5s;
            /* Number of times to run */
            animation-iteration-count: 3; /* or infinite */
            /* Whether to alternate direction */
            animation-direction: alternate; /* or normal, reverse, alternate-reverse */
            /* What styles to apply outside of the animation */
            animation-fill-mode: forwards; /* retains the styles of the last keyframe */
            animation-play-state: running; /* or paused */
        }

        /* Shorthand property */
        .animated-shorthand {
            animation: fadeIn 1s ease-in 0.5s 3 alternate forwards;
        }

Gotchas & Best Practices:

  • Use transform for performance-intensive animations, as it’s often hardware-accelerated.
  • Be mindful of transform-origin when rotating or scaling.
  • Transitions are for simple, state-based changes; animations are for complex, multi-step sequences.
  • Keep animations subtle to avoid distracting the user.

Responsive Design

Media Queries

Apply styles based on device characteristics.

/* Basic media query for screens wider than 768px */
        @media screen and (min-width: 768px) {
            .container {
                width: 750px;
            }
        }

        /* Complex media queries */
        @media (min-width: 768px) and (max-width: 1024px) {
            /* Styles for tablets */
        }

        @media (orientation: landscape) {
            /* Styles for landscape mode */
        }

        @media (prefers-color-scheme: dark) {
            body {
                background-color: #333;
                color: white;
            }
        }

Meta Viewport Tag

Essential for mobile-first responsive design.

<meta name="viewport" content="width=device-width, initial-scale=1.0">
  • width=device-width: Matches the viewport width to the device width.
  • initial-scale=1.0: Sets the initial zoom level when the page is first loaded.

Responsive Units

Use units that scale with the viewport.

.responsive-text {
            font-size: 2vw; /* 2% of the viewport width */
        }

        .fluid-container {
            width: 80vw; /* 80% of the viewport width */
            height: 50vh; /* 50% of the viewport height */
        }

        .dynamic-font-size {
            /* scales from 1rem to 2rem based on viewport width */
            font-size: clamp(1rem, 2vw + 1rem, 2rem);
        }

Gotchas & Best Practices:

  • Adopt a mobile-first approach by writing styles for smaller screens first, then using min-width media queries to add styles for larger screens.
  • Avoid using max-width on the viewport meta tag.
  • Test your designs on actual devices or with browser developer tools to ensure they work as expected.

Gotchas & Best Practices:

  • Box shadows can be performance-intensive, especially on mobile devices; use them sparingly.
  • opacity affects the entire element, including its children. To change only the background’s opacity, use an rgba or hsla color.
  • filter properties can be combined to create unique visual effects.
